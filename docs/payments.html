Payment Integration

Set up secure payment processing for your Chrome extension with Stripe, PayPal, licensing servers, and more.

Stripe Integration

Stripe is the most popular payment processor for Chrome extensions. It offers robust APIs, excellent developer experience, and supports subscriptions, one-time payments, and more.

💡 Pricing Strategy Tip

Start with annual billing at 2x the monthly price. Stripe data shows annual subscriptions reduce churn by 30-40% and increase LTV significantly.

Setup Steps

  1. Create a Stripe Account

    Sign up at stripe.com and complete account verification.

  2. Get API Keys

    Navigate to Developers → API Keys. You'll need the publishable key (for frontend) and secret key (for backend).

  3. Create Products and Prices

    In the Stripe Dashboard, create your products (subscription plans, one-time purchases) and get their price IDs.

  4. Set Up Webhooks

    Configure webhooks to receive payment events (checkout completed, subscription updated, payment failed).

Frontend Integration

<script>
  const stripe = Stripe('pk_test_your_publishable_key');
  
  document.getElementById('checkout-btn')
    .addEventListener('click', async () => {
      const response = await fetch('/create-checkout-session', {
        method: 'POST',
      });
      const session = await response.json();
      
      const result = await stripe.redirectToCheckout({
        sessionId: session.id,
      });
      
      if (result.error) {
        console.error(result.error.message);
      }
    });
</script>

    payment_method_types: ['card'],

      price: 'price_1234567890',
      quantity: 1,
    }],
    mode: 'subscription',
    success_url: 'https://yoursite.com/success',

    }

});

Pro Tip: Use Stripe Customer Portal to let users manage their subscriptions (cancel, upgrade, update payment method) without building your own UI.

PayPal Integration

PayPal remains popular, especially in certain regions. It offers both PayPal Checkout and subscription APIs.

PayPal Checkout Popular

Standard PayPal button integration. Good for one-time payments and simple subscriptions.

PayPal Subscriptions

Full subscription management with billing agreements. Supports complex billing scenarios.

Setup Steps

  1. Create PayPal Developer Account

    Sign up at developer.paypal.com.

  2. Create App

    Create a new app in the Dashboard to get your Client ID and Secret.

  3. Add JavaScript SDK

    Include the PayPal SDK in your extension's options page or website.

paypal-button.js<script src="https://www.paypal.com/sdk/js?client-id=test"></script>

<div id="paypal-button-container"></div>

<script>
  paypal.Buttons({
    createOrder: (data, actions) => {
      return actions.order.create({
        purchase_units: [{
          amount: { value: '9.99' },
          description: 'Pro Plan - My Extension'
        }]
      });
    },
    onApprove: async (data, actions) => {
      const order = await actions.order.capture();
      // Send to your server for verification
      await fetch('/verify-paypal-payment', {
        method: 'POST',
        body: JSON.stringify(order)
      });
    }
  }).render('#paypal-button-container');
</script>

Note: PayPal subscriptions require additional setup for webhook verification. Always validate payments server-side.

Chrome Web Store Payments Deprecated

Google deprecated Chrome Web Store payments in 2020. Existing subscriptions were migrated, but new extensions cannot use CWS billing.

Important: If you have legacy CWS subscriptions, migrate to Stripe or another provider as soon as possible. Google may fully remove support.

Migration Considerations

  • No new sign-ups: CWS payments only work for existing subscribers
  • Migration path: Offer existing users a discounted rate to switch
  • Data export: Contact Google for subscriber data if needed
  • Communication: Email users well in advance of any changes

💡 Pricing Strategy Tip

When migrating from CWS, offer a "founding member" discount (20-30% off) to early supporters who switch. This builds loyalty and reduces churn.

License Key Servers

License key servers allow you to sell perpetual licenses or manage subscription licenses with your own activation system. Popular options include Gumroad, LemonSqueezy, and Keygen.

LemonSqueezy Recommended

All-in-one payments platform with built-in license key generation, subscriptions, and global tax handling. Great Stripe alternative.

Gumroad

Simple product sales with license keys. Good for one-time purchases. Takes a higher cut but minimal setup.

Keygen

Developer-focused license management with powerful API. Best for complex licensing rules and custom implementations.

LemonSqueezy Setup

  1. Create Store

    Sign up at lemonsqueezy.com and create your first store.

  2. Create Product

    Add your extension as a product with pricing (one-time or subscription). Enable license keys.

  3. Configure Webhooks

    Set up webhooks to receive license creation events and subscription updates.

  4. Implement License Check

    Add license validation in your extension's background script or options page.

license-check.js - Validate Licenseasync function validateLicense(licenseKey) {
  const response = await fetch('https://api.lemonsqueezy.com/v1/licenses/validate', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${your_api_key}`,
      'Content-Type': 'application/vnd.api+json',
      'Accept': 'application/vnd.api+json'
    },
    body: JSON.stringify({
      license_key: {
        key: licenseKey
      }
    })
  });
  
  const data = await response.json();
  return {
    valid: data.data.attributes.valid,
    expiresAt: data.data.attributes.expires_at,
    customerEmail: data.data.attributes.customer_email
  };
}

Security Tip: Never store license keys in plain text. Hash them and validate on your server before enabling features.

💡 Pricing Strategy Tip

For license keys, consider a "two-tier" pricing: basic (single device) and pro (unlimited devices). This captures more revenue from power users.

// Handle upgrade button
document.getElementById('upgrade-btn').addEventListener('click', async () => {
  const session = await createCheckoutSession('monthly');
  
  // Open payment in new tab
  window.open(session.url, '_blank');
  
  // Poll for payment completion
  pollForPayment(session.id);
});

async function pollForPayment(sessionId) {
  const checkInterval = setInterval(async () => {
    const status = await checkPaymentStatus(sessionId);
    if (status === 'completed') {
      clearInterval(checkInterval);
      await storeLicenseKey(status.licenseKey);
      showPremiumFeatures();
      // Optionally reload extension or notify background
    }
  }, 3000);
}

Best Practices

  • Graceful degradation: Show clear upgrade prompts but let users try core features first
  • Transparent pricing: Clearly display what's included in free vs paid
  • Easy restoration: Implement "Restore Purchases" for users who reinstall
  • Secure storage: Store license info encrypted, never in plain text

Success Pattern: Extensions with "freemium" models (free tier + paid upgrade) convert 3-5% of users, vs 1-2% for paid-only.
